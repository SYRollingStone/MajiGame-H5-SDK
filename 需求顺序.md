# MajiGame H5 SDK 需求开发顺序

> 项目目标：为 [majigame.com](https://majigame.com) 自营 H5 小游戏站构建一套类似 Poki / CrazyGames 的游戏 SDK，供站内所有第三方/自营游戏接入。
>
> 部署架构：游戏前端 + SDK 部署于 Cloudflare Pages；后端逐步迁移至 Cloudflare Workers（D1 / KV / R2 / Analytics Engine）。
>
> 文档维护规则：每完成一期，在对应章节末尾标注 ✅ 完成时间；变更需求时回到本文件修订，禁止零散更新。

## 第 1 期：安全防护模块（本期）

### 1.1 目标
做到"4399 同款体验"：玩家打开 F12 → 调试器卡住 + 全屏遮罩提示"请关闭开发者工具" → 玩家关掉 F12 后游戏自动恢复。同时阻断常见的快捷键和右键查看源码，限制 SDK 只能在白名单域名下运行。

### 1.2 功能清单

#### A. 快捷键 / 右键阻断（`security/shortcuts`）
- 拦截 `F12`、`Ctrl+Shift+I`、`Ctrl+Shift+J`、`Ctrl+Shift+C`、`Ctrl+U`、`Ctrl+S`
- 拦截 `contextmenu`（右键菜单）
- 拦截文本选中 / 拖拽
- 支持开关、支持给特定 DOM 选择器放行（游戏内输入框等）

#### B. 反调试检测（`security/anti-debug`）
5 类独立检测器，并联使用，命中任一即触发：
1. `debugger` 陷阱循环（高频）
2. `performance.now()` 时间差检测（中频）
3. `console.log` getter 副作用检测
4. `window.outerHeight / outerWidth` 尺寸差检测
5. `Function.prototype.toString` 重写检测

#### C. 命中后默认动作（`freeze + tip`）
- 持续触发 `debugger` 卡死调试器
- 显示全屏遮罩，提示文案可配置
- 检测到 DevTools 关闭后**自动撤遮罩、退出循环、游戏恢复**
- 备用动作：`report` / `redirect` / `blur` / `reload` / `noop`

#### D. 域名 / 来源锁定（`security/domain-lock`）
- `location.hostname` 白名单校验（支持 `*.majigame.com` 通配）
- `document.referrer` 校验
- 命中后动作复用 anti-debug 的策略

### 1.3 对外 API（草案）

```html
<script src="https://cdn.majigame.com/sdk/maji-sdk.min.js"></script>
<script>
  MajiSDK.init({
    gameId: 'cat-jump',
    security: {
      antiDebug: {
        enabled: true,
        onDetect: 'freeze+tip',
        tipText: '检测到您打开了开发者工具，请关闭后继续游戏',
        interval: 1500,
        detectors: ['debugger', 'timing', 'console', 'size', 'toString'],
      },
      blockShortcuts: true,
      blockContextMenu: true,
      allowedHosts: ['majigame.com', '*.majigame.com', 'localhost', '127.0.0.1'],
      allowedReferrers: ['majigame.com'],
    }
  })
</script>
```

### 1.4 防护边界（坦白说）
- **能挡住**：好奇玩家、随手 F12 看源码、复制小白、普通脚本党
- **挡不住**：专业逆向、Charles/Fiddler 抓包、外接 Chrome DevTools Protocol、浏览器扩展直接读 Sources
- **真正的护城河**：核心校验（分数提交签名、付费校验、关键随机数）必须放服务端 Worker（第 5/8 期实现）

## 第 2 期：全链路打通（本期）

### 2.1 目标
SDK 写完后，必须走通 **打包 → CDN 发布 → 第三方游戏项目通过 `<script src>` 引入 → 调用 SDK API → 验证功能正常** 的完整链路。这一期不是"自测能跑就行"，而是真正模拟接入方的工作流。

### 2.2 任务清单
1. **打包配置**：Vite 库模式产出 UMD 格式 `maji-sdk.min.js`
2. **混淆配置**：javascript-obfuscator 后处理，控制流平坦化 + 字符串数组加密 + 自我保护
3. **本地构建验证**：`npm run build` 产出无报错，文件体积合理（首期 ≤ 100KB gzip）
4. **示例 demo**：`examples/security-demo.html` 直接 `<script src="../dist/maji-sdk.min.js">` 引入并跑通
5. **独立接入测试项目**：在 SDK 仓库外另建一个最简 HTML 项目，模拟"第三方游戏接入"场景，验证：
   - CDN 引入方式可用
   - `MajiSDK.init()` 配置生效
   - 打开 F12 触发 freeze+tip，关闭恢复
   - 右键 / 快捷键被拦截
   - 非白名单域名访问被阻断
6. **CDN 发布预案**：本期暂不真正上传 Cloudflare，但要明确发布路径和文件命名规范

### 2.3 产物文件
- `dist/maji-sdk.min.js`（UMD，浏览器全局变量 `MajiSDK`）
- `dist/maji-sdk.esm.js`（保留，为以后 npm 发布预留）
- `dist/maji-sdk.d.ts`（TypeScript 类型声明）

## 第 4 期：数据分析前端模块（待开始）

- `MajiSDK.analytics.track(event, payload)` 统一 API
- 事件队列 + 批量发送（5 秒 or 20 条 flush）

- 失败重试 + localStorage 暂存

## 第 5 期：Cloudflare Workers 后端（独立仓库）

新建仓库 `majigame-backend`，使用 [Hono](https://hono.dev) 框架（跨平台，方便以后迁移）。

- `POST /api/track`：接收 analytics 事件 → 写 Analytics Engine
- 域名 / 签名校验
- 限流
- adapter 抽象层：未来可平迁到 Node + PostgreSQL
- Cron Trigger：每日把 AE 数据备份到 R2（迁移护城河）
